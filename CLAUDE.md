# 프로젝트 지침

## 트러블슈팅 자동 기록

작업 중 **버그·에러·막힘을 해결했다고 판단되면, 사용자가 따로 요청하지 않아도**
`log-trouble` skill을 사용해 `.troubleshooting/TROUBLESHOOTING.md`에 기록한다.

- 기록 후에는 "트러블슈팅에 기록해뒀어요 (제목)" 한 줄로 알린다.
- 사소한 오타·문법 오류 등은 기록하지 않는다. **의미 있는 문제**만 (환경설정, 통합,
  배포, 비동기/상태 버그, 외부 API/CORS, 인증 등 다시 만날 법한 것).
- 같은 문제가 이미 기록돼 있으면 새로 만들지 말고 기존 항목을 보완한다.
- 민감 정보(키·토큰·비밀번호)는 마스킹하거나 제외한다.
- `.troubleshooting/`는 `.gitignore` 대상이라 git에 올라가지 않는다 (개인 기록).
