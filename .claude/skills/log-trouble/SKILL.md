## 목적

겪은 문제와 해결 과정을 `.troubleshooting/TROUBLESHOOTING.md` 한 파일에
계속 누적 기록한다. 같은 문제를 또 만났을 때 빠르게 찾아보기 위함.

## 저장 위치

- 파일: `.troubleshooting/TROUBLESHOOTING.md`
- 이 디렉토리는 **`.gitignore`에 포함**되어 git에 올라가지 않는다 (개인 기록).
- 디렉토리/파일이 없으면 자동 생성한다.
- `.gitignore`에 `.troubleshooting/`가 없으면 자동으로 추가한다.

## 기록 방식

- 새 기록은 파일 **맨 위(최신순)** 에 추가한다. (헤더 바로 아래)
- 한 문제 = 한 블록. 기존 내용은 절대 덮어쓰지 않는다.
- 날짜/브랜치는 직접 조회해서 채운다:
  - 날짜: 시스템 현재 날짜 (예: 2026-06-07)
  - 브랜치: `git branch --show-current`

## 기록 항목

| 항목 | 설명 |
| --- | --- |
| 제목 | 문제를 한 줄로 |
| 날짜 / 브랜치 | 발생 날짜, 작업 브랜치 |
| 증상 | 무슨 일이 일어났는지 |
| 원인 | 왜 생겼는지 (파악된 경우) |
| 해결 | 어떻게 고쳤는지 (핵심) |
| 에러 로그 / 코드 | 실제 에러 메시지, 관련 코드 스니펫 |
| 재발 방지 / 참고 | 다음에 안 겪으려면 + 참고한 문서/이슈 URL |
| 관련 파일 | 영향받은 파일 경로 |

## 기록 템플릿

````markdown
## [{날짜}] {제목}

- **브랜치:** {branch}
- **관련 파일:** `path/to/file.ts`, `path/to/other.ts`

### 증상
{무슨 일이 일어났는지}

### 원인
{왜 생겼는지}

### 해결
{어떻게 고쳤는지 — 핵심}

### 에러 로그 / 코드
```
{실제 에러 메시지 또는 코드 스니펫}
```

### 재발 방지 / 참고
- {다음에 안 겪으려면}
- 참고: {문서/이슈 URL}

---
````

## 실행 순서

1. 파일/디렉토리 존재 확인 → 없으면 생성 (헤더 `# 트러블슈팅 로그` 포함)
2. `.gitignore`에 `.troubleshooting/` 없으면 추가
3. 현재 날짜와 `git branch --show-current`로 브랜치 조회
4. 대화 맥락에서 증상·원인·해결·에러·관련파일을 정리
5. 빠진 항목이 있으면 사용자에게 간단히 물어본다 (전부 다 캐묻지 말 것)
6. 템플릿대로 작성해 파일 **맨 위**에 삽입
7. 저장 후 추가된 항목 제목과 파일 경로를 출력

## 주의사항

- 민감 정보(키, 토큰, 비밀번호)는 마스킹하거나 제외한다.
- 사용자가 명시적으로 요청할 때만 기록한다 (자동 기록 금지).
- 해결되지 않은 문제도 기록 가능 — 그 경우 "해결" 대신 "현재 상태/시도한 것"을 적는다.
