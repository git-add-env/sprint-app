## 사용법

사용자가 기능이나 버그를 설명하면:

1. 설명을 분석하여 이슈 제목, 본문, 라벨을 결정한다
2. 사용자에게 내용을 확인받는다
3. `gh issue create`로 이슈를 생성한다
4. 생성된 이슈 번호와 URL을 출력한다

## 라벨 매핑

| 내용        | 라벨       |
| ----------- | ---------- |
| 새 기능     | `feat`     |
| 버그 수정   | `fix`      |
| 스타일      | `style`    |
| 리팩토링    | `refactor` |
| 설정/의존성 | `chore`    |
| 문서        | `docs`     |

## 실행

```bash
gh issue create \
  --title "{type}: {제목}" \
  --body "{본문}" \
  --label "{라벨}" \
  --assignee "@me"
```

## 이슈 본문 템플릿

```markdown
## 설명
{사용자가 설명한 내용 정리}

## 작업 항목
- [ ] 항목 1
- [ ] 항목 2
```

## 주의사항

- `gh` CLI 인증 필수
- 라벨이 존재하지 않으면 라벨 없이 생성하고 안내
- 생성 전 반드시 사용자 확인
